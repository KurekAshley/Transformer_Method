## Versioning Strategy

### Major Version

A major version change occurs when:

- Core doctrine changes
- Foundational frameworks change
- Repository architecture changes significantly

Example:

```text
1.0.0 → 2.0.0
```

---

### Minor Version

A minor version change occurs when:

- New doctrine files are added
- New categories are introduced
- Existing doctrines are expanded

Example:

```text
1.0.0 → 1.1.0
```

---

### Patch Version

A patch version change occurs when:

- Grammar corrections are made
- Formatting improvements are made
- Diagrams are improved
- References are updated

Example:

```text
1.0.0 → 1.0.1
```
